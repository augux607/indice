Web Workers

web Workesr. MDN WEB DOC

Los Web Workers hacen posible ejecutar la operación de un script en un hilo en segundo plano separado de la ejecución el hilo principal de la aplicación web. La ventaja de esto es que un proceso laborioso puede actuar en un hilo separado, permitiendo al hilo principal (normlamente la UI) ejecutarse sin ser bloqueado o ralentizado.

Concepto y uso

Un worker es un objeto que se crea a través del constructor ( Worker() ) este ejecuta un archivo javascript — este fichero contiene los scripts que se van a ejecutar en el hilo del worker; los workers se ejecutan en un contexto global diferente al actual del navegador window. Este contexto está representado por el objeto DedicatedWorkerGlobalScope en el caso de los workers dedicados (los workers standard utilizan un solo script; y los shared workers usan SharedWorkerGlobalScope).

ATRIBUTOS DE ENTRADA EN HTML

A continuaciòn, un ejemplo del uso de los diferentes atributos de entrada en html

  1. El atributo -value- especifìca un valor inicial para un campo de entrada: input type="text" id="name" name="name" value="Cesar"

  2. El atributo -multiple- especifica que el usuario puede ingresar más de un valor en un campo de entrada: input type="file" id="files" name="files" multiple

  3. El atributo -pattern- especifica una expresión regular con la que se verifica el valor del campo de entrada cuando se envía el formulario:input type="file" id="files" name="files" multiple El atributo pattern funciona con los siguientes tipos de entrada: texto, fecha, búsqueda, url, tel, correo electrónico y contraseña.

  4. El atributo -placeholder- especifica una pista breve que describe el valor esperado de un campo de entrada (un valor de muestra o una breve descripción del formato esperado): input type="tel" id="telefono" name="telefono"placeholder="304-5316565". La pista breve se muestra en el campo de entrada antes de que el usuario ingrese un valor. El atributo placeholder funciona con los siguientes tipos de entrada: texto, búsqueda, url, tel, correo electrónico y contraseña.

  5. El atributo -required- especifica que se debe completar un campo de entrada antes de enviar el formulario: input type="text" id="username" name="username" required="required" . El atributo required funciona con los siguientes tipos de entrada: texto, búsqueda, url, teléfono, correo electrónico, contraseña, selectores de fecha, número, casilla de verificación, radio y archivo.

  6. El atributo -autofocus- especifica que un campo de entrada debería enfocarse automáticamente cuando se carga la página: input type="text" id="fname" name="fname" autofocus . A continuacòn un ejemplo para enfocar en el campo de entrada nombre cuando se carga la pàgina

    Datos Usuario

  7. El atributo -autocomplete- especifica si un formulario o un campo de entrada debe tener autocompletado activado o desactivado: form action="quibit.php" autocomplete="on" . Autocompletar permite al navegador predecir el valor. Cuando un usuario comienza a escribir en un campo, el navegador debe mostrar opciones para completar el campo, en función de los valores escritos anteriormente. El atributo autocomplete funciona con -form-y los siguientes -input- tipos: texto, búsqueda, url, tel, correo electrónico, contraseña, fechadores, rango y color.

FORMULARIOS EN HTML

Un formulario se utiliza principalmente para obtener informaciòn de entrada del usuario.

Elemento Descripciòn
form El elemento < form > HTML define un formulario que se utiliza para recopilar la entrada del usuario
input El elemento < input > HTML define defne los tipos de entrada de datos del usuario y estos pueden ser de varios tipos.
Ejemplos: -input type = "text" :Define un campo de entrada de texto de una sola línea.-
input type = "radio": Define un botón de radio (para seleccionar una de muchas opciones)
input type = "submit": Define un botón de envío (para enviar el formulario)

El atributo action de un formulario define la acciòn que se realizarà al enviar un formulario. Para el siguiente ejemplo este valor es: action="index.html target="_blank" method="get"

El atributo target especifica si el resultado enviado se abrirá en una nueva pestaña del navegador, un marco o en la ventana actual. El valor predeterminado es " _self", lo que significa que el formulario se enviará en la ventana actual. Para que el resultado del formulario se abra en una nueva pestaña del navegador, use el valor " _blank".

El atributo method especifica el método HTTP ( GET o POST ) que se utilizará al enviar los datos del formulario.

ETIQUETAS SEMÀNTICAS EN HTML

Son las etiquetas utilizadas y que dan significado a las partes de un documento. Es decir, el nombre de la etiqueta permite determinar su contenido y son entendible tanto por los navegadores como por los desarrolladores.

En desarrollo web, las etiquetas HEADER, ARTICLE, SECTION, FOOTER, ETC., sirven para definir el esquema principal del documento, estas etiquetas indican què es el contenido que engloban y cual es su relaciòn con el conjunto de elementos del documento HTML.

Existe pues una forma lógica de colocar todos los contenidos dentro de un sitio y unas etiquetas determinadas para contenerlos, atendiendo a la estructura que nosotros queremos comunicar a los sistemas que puedan procesar el documento.

Nota: Realmente, las etiquetas estructurales están ahí y nosotros somos los que podemos utilizarlas a nuestro gusto, siempre que respetemos la significación que se les ha dado. Hay discusiones de sobra para interpretarlas de una manera o de otra y para decir cómo deberían ser utilizadas. Podemos usar el sentido común para imaginar lo que pueda o no ser correcto.

CONFIGURACIÒN DE LA VENTANA GRÀFICA

En HTML existe una forma para poder tener el control de la ventana gràfica a travès de la etiqueta < meta >

la ventana gràfica es el àrea visible del usuario de una pàgina web. Para ello se debe incluir la etiqueta < meta > en todas las pàginas web quw se puedan construir:

Un elemento -meta- de ventana gràfica proporciona al navegador instrucciones sobre como controlar las dimensiones y las escalas de la pàgina.

La parte width = device-width establece el ancho de la pàgina para seguir el ancho de la pantalla del dispositivo (que variarà segùn el dispositivo).

La parte initial-scale = 1.0 establece el nivel de zoom inicial cuando el navegador carga por primera vez la página.

Esto es muy importante para la construcciòn de pàginas web responsive, debido a que se le darà la instrucciòn a los navegadores acerca de còmo controlar las dimensiones de la pàgina de acuerdo al dispositivo donde se vea la pàgina web.
